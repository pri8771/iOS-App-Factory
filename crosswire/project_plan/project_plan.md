That distinction matters twice: for deterministic testing (if it's randomized, the mock-transport test suite needs a seeded/injectable RNG or every test run produces different fragment assignments and "the same test failed differently" becomes a recurring debugging tax), and for the replay flow's actual feel (does replaying the same pack redeal fragments differently, which is presumably the point of replay value, or does it always deal the same way).
