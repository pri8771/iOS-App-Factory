## Final Output

**Architecture overview:** Single Xcode project (`Somna.xcodeproj`), three targets — `Somna` (app), `SomnaWidget` (WidgetKit extension, also hosts the Live Activity), `SomnaTests` (Swift Testing unit tests). No SPM-local-package split; one app target with clean internal folders gives full testability without build-graph overhead. `@Observable` view models throughout (iOS 17+, no Combine). Two repository protocols (`SleepSessionRepository`, `CheckInRepository`) injected at the View layer via `@Environment` and passed into view-model initializers as concrete protocol instances — view models never pull `@Environment` themselves, so every view model is constructible in tests with in-memory fakes. The correlation engine is a real `actor` operating only on plain `NightRecord` structs, never SwiftData `@Model` objects, which is what keeps the non-negotiable threshold-gating test (N‑1 nights → zero scored insights, N nights → at least one) a pure unit test with no `ModelContainer`. Zero third-party dependencies — every capability (persistence, subscriptions, background activity, health read access, sharing, comparison bars) has a first-party Apple framework.

**File/module layout:**
- `App/` — `@main` App struct, scene setup, root `TabView`.
- `Models/` — the two `@Model` types plus plain-struct value types the engine consumes.
- `Repositories/` — `SleepSessionRepository`/`CheckInRepository` protocols; `Repositories/SwiftData/` for the real implementations; in-memory fakes live in the test target.
- `Engine/` — `CorrelationEngine` actor, `InsightRule` implementations, `InsightResult`/`InsightCardData`, tag/energy enums, `NightRecordAssembler`, and the pure `gradient(for:)` function (treated as derived-state logic even though it feeds visuals).
- `ViewModels/` — one `@Observable` class per screen: `TodayViewModel`, `InsightsViewModel`, `HistoryViewModel`, `RecapViewModel`, `PaywallViewModel`.
- `Views/` — grouped by screen per design_handoff's component list (`Today/`, `Insights/`, `History/`, `Recap/`, `Settings/`, `Paywall/`, `Components/` for `FrostedCard`/`InsightCard`/`EnergyScaleControl`/etc.).
