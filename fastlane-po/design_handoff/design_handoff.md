Raw OCR and Raw JSON tabs are read-only monospace panels — don't over-design these, they're debugging views for power users, but they do need copy-to-clipboard and should render even when the JSON is the invalid pre-repair version so ops can see what the repair prompt fixed.

Density: this is a professional tool used by AP clerks for hours a day, so default to compact spacing in tables (36–40px row height) rather than the airy SaaS-marketing-site spacing that looks nice in a screenshot and is exhausting in real use.

Accessibility isn't optional garnish here: confidence must never be color-only — every low-confidence field needs a text/icon indicator (a small warning glyph + "62%" label) alongside the amber border, because color-blind users and anyone in bright light need a non-color signal, and this is exactly the kind of finding an auditor would flag first. Motion should be near-invisible: 120–150ms ease for panel transitions, a subtle highlight-pulse (not a bounce) when a field is programmatically scrolled-to from a PDF click, and stepper-dot state changes on the upload screen should animate in under 200ms — nothing here should feel playful, this is closer to a cockpit instrument than a consumer app.
