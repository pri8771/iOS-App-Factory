## Final Output

**Target user and use case:** The host who taps "create room" for an in-person gathering of 10-12 people (game night, family event, dorm/office social), who wants zero setup friction and zero mid-game disputes. Joiners just need to get in fast (App Clip or manual room code) and trust the game once seated. This is a "hosted group play" niche, not a mass-casual download target.

**Core loop:** Host creates room and confirms local-join consent → players join via App Clip or manual code and take seats → host locks seats and starts → draft phase (players actually pick/pass cards under enforced rules — the app referees, it does not play for them) → trick phase repeats (play a legal card, trick resolves, winner leads next) → round scores post → session settles → replayable command log persists locally so anyone reconnecting can resync. The trick-play turn is the highest-scrutiny screen: whose turn, what's legal, did it register has to be instantly unambiguous, with the seam states between phases (mid-join, timed-out play, host battery/health warning, host-lost/migration, draft paused for reconnect) treated as first-class named UI states, not edge cases.

**Paid value and subscription value:** A one-time, non-consumable **Host Pass** purchase — not a subscription — because game nights are occasional and recurring billing punishes exactly this usage pattern. The free tier is the complete, ungated 10-12-player live game with the default Spades-style ruleset — no player-count gating, no time-limited trial, no watered-down "demo" core. Host Pass unlocks real, non-cosmetic utility built on infrastructure the reliability spine already requires: additional enforced rulesets (Hearts-style, Euchre-style), a replay/dispute viewer that scrubs the signed command log trick-by-trick, host-visible session telemetry and an adjudicator view (per-player connection quality, host battery/load, live "what just happened" resolution), and cross-session group history/stats/tournament tracking with carried standings across nights. Before this ships, the replay viewer must pass its own fidelity test — replaying the log and asserting it reconstructs the exact state that was actually observed live, across the same host-kill, disconnect-mid-draft, and packet-reordering scenarios the reliability contract already requires.

**Competitive wedge:** Trestle sells a receipt, not just a ruleset. The signed, replayable, host-authoritative command log is both what makes 10-12-player sync trustworthy and the literal basis of the premium replay/adjudication feature — competitors are "works most of the time," Trestle is "works, and can prove it," even when phones misbehave.

**Viral/niche growth angle:** Growth is host-to-host, earned by the product working under pressure, not gimmicks. App Clip join means every joiner experiences the polished app before installing anything, and a good chunk go on to host their own night. Saved presets (player list, seating, ruleset) make repeat hosting one tap, which is what turns a single good night into "this is just how our group plays cards now." No artificial invite bonuses or streaks — the story ("we played and nobody argued") is the marketing.

**Local-first and cloud-ready plan:** No cloud dependency, no required account, all state (command log, sessions, replays) stored on-device. Cloud-readiness is preserved structurally, not bolted on: versioned command-log schema, transport-agnostic design, and entitlement/ruleset fields already modeled in the data layer, so optional iCloud sync or backup later is a storage/transport swap, not an engine rewrite.
