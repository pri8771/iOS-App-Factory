States — loading: cosmetic pending-tap; empty: trick led but nobody's played yet; success: card animates into its seat position; error: `move_rejected_illegal` ("not legal — follow suit," offending card outlined) vs. `move_rejected_stale_trick` ("this trick already moved on," non-blaming tone) as genuinely distinct copy and tone.
- *Round Transition:* full-bleed takeover — round score recap as hero content, restrained scale/fade (no page-turn/card-flip, that's reserved for actual card motion) into "Round 2 — Dealer: [name]" with the new dealer visually marked.
- *Match End:* one screen, two zones — cumulative score as hero content (top two-thirds), "one-round-pair session in v1" banner as a smaller, visually secondary strip below. Defensive fallback state for the (architecturally-shouldn't-happen) incomplete-score case rather than a crash or blank screen.
- *Cross-cutting recovery states:* amber "connection at risk" banner rendered as a non-blocking top-anchored overlay above whatever screen is current (not a route swap, so it never destroys in-progress screen context); hard "disconnected — host is deciding" state renders as a translucent scrim + overlay, host-only action sheet (continue/skip/abort) for the host, passive waiting card for joiners — one shared overlay layer so it reads as one event, not two coincidental screens.

**Design tokens:** `feltGreen` (`#1C3A2E`) as the default game-surface color (chosen over walnut for unambiguous "card table" read); warm off-white card surface (`#F7F1E6`, never pure white); single reserved accent `#E0A83E` (warm amber/gold) used exclusively for turn indication, error outlines, and pending-decision states — never decoratively. Typography: SF Pro Rounded for game-facing text, standard SF Pro for host instrumentation numerals (reinforcing the "different job" distinction). Motion: spring transitions (response ~0.35-0.45, damping ~0.8) for card placement, linear/ease for banners, explicit avoidance of confetti/particles/bounce at Match End given the honest "more is coming" framing.

**Component inventory:** `SeatGridView` (shared Lobby/Waiting Room), `HostStatusStripView`, `CardView` (single unified renderer for hand + trick-in-progress cards, parameterized by legality/pending/played state — protected against a second context-specific card renderer ever forking off), `TurnRingModifier`, `RejectionToastView` (two copy variants), `RecoveryOverlayView`, `ScoreRecapView` (shared Round Transition/Match End), `RoomIdentityCardView`.

**Accessibility:** combined VoiceOver phrase per trick-table/draft element stating card identity, legality, and turn state together ("Play Queen of Hearts, your turn, legal"); focus order follows seat/play order for trick history then moves to your own hand, not raster order. Dynamic Type capped at the trick table specifically (geometry-driven ceiling, not an app-wide cap) while score/instruction/banner screens scale freely.

**Claude Design decision:** No external design pass. This app's interaction fidelity — seat-relative geometry as a function of live player count, haptic timing tied to command-log state transitions, a cosmetic pending-state pattern reconciled against real network latency, and rejection copy that only makes sense in the context of the state machine producing it — doesn't survive a round-trip through static comps. This spec goes straight to the SwiftUI build phase as tokens + component specs, not Figma/Claude-Design mockups.

Codex's absence is logged as an infrastructure gap across both rounds, not a design disagreement. Gemma3's repeated push to drop App Clip, collapse to single-round, and simplify the rejection messaging is logged again as non-engagement with this phase's actual scope, consistent with its position across seven prior phases — none of it is adopted, since all of it re-argues points already closed in app_features and design_discussion.
