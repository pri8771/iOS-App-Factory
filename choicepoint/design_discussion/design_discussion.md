Empty Ledger → tap "+" → three-field capture (decision, prediction, revisit date) → save in seconds → sheet dismisses, entry appears in Ledger under Upcoming with entry number and due date. Weeks later, a discreet notification ("1 revisit due") arrives; tapping it deep-links straight into that decision's Revisit screen, which opens directly onto the stamped Before block — no gate, nothing to unlock. User writes what happened and taps held/missed/mixed (resolving, freeing a cap slot, updating Calibration live) or taps the secondary "not yet" (only pushes the date forward, decision stays active). Once at least one decision is resolved, Calibration shows real counts instead of an empty state; confidence-bucketed analytics stay behind a locked, honestly-labeled teaser until upgrade. Hitting the 5-active free cap surfaces inline in Capture before save is attempted, routing the user either toward resolving an existing overdue item or toward the contextual paywall.

**State model per screen**
- **Ledger**: first-run empty / no-active empty / populated active / filtered resolved / permission-warning visible / at-cap banner / persistence-error.
- **Capture**: pristine / partially entered / valid-savable / saving / cap-blocked (shown before save attempt) / validation error / save failure / save success.
- **Decision Detail**: loading-by-id / active-loaded / resolved-loaded / stale-or-deleted error.
- **Revisit/Resolve**: loading / due / overdue-with-day-count / ready (Before block shown, outcome actions available) / submitting outcome / resolved-success / reschedule-success (not-yet) / already-resolved-elsewhere fallback (read-only) / submission failure.

- **Onboarding**: pre-permission explanation / granted / denied / skipped.
- **Settings**: normal / notifications off-or-revoked / paid-active / paid-lapsed-over-cap / export running/failed.
- **Widgets/Siri**: success / cap-blocked / validation failure / open-app fallback.

Paper-and-ink ledger aesthetic: warm off-white background, near-black ink text, hairline rules between rows instead of shadowed cards, zero gradients or bouncy/celebratory animation. One sparse accent color (deep brass or oxblood), used only for the compose action and overdue emphasis; overdue is signaled primarily through weight/placement/text, never color alone.

**Accessibility notes**
Dynamic Type must reflow every ledger row and the Before block without truncating decision text, prediction, or date at accessibility sizes — tested at largest text size before visual design is considered final, given how much of the design leans on serif/tabular typographic contrast. VoiceOver reads each ledger row as one coherent sentence (entry number, decision, revisit date, status), not fragmented elements. Held/missed/mixed and overdue status never rely on hue alone — always paired with explicit text or a glyph. Widget summaries are plain accessible sentences ("2 decisions overdue this week").
