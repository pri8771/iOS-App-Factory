Non-color match feedback (outline + numeric overlap + haptic tick), full VoiceOver labeling, plus a mandatory discrete non-gesture control path for VoiceOver users (select object, nudge arrows, coarse/fine rotation stepper, explicit duplicate/delete/reset controls, accessibility announcements of numeric overlap) driven through the same underlying mutation functions as the drag gesture — never a forked control/data model. Determinism is checked two distinct ways: epsilon tolerance (~1e-8) on live geometry/solve replay, and exact structural equality on canonical serialized snapshots at save/load — never a literal "bitwise-identical" claim on floats. Share transport is a plain artifact string via the system Share Sheet as the primary path; any future URL-scheme/deep-link import is just another untrusted input into the identical three-state decoder, never a trusted fast path. **Edge cases:** zero-area/near-zero occluders; touching-only overlap; duplicate occluders at near-equal bounds; 1px/edge-aligned target intersections; light source exactly on or just inside/outside an occluder boundary; rotation wraparound (359°→0°) normalization consistency; large drag-delta jitter; simultaneous move+rotate gesture interrupted mid-flight (resolved: roll back to last committed transform, never persist a frozen partial gesture); app backgrounded mid-gesture, mid-export, or mid-import; force-quit mid-edit; stale/partial local saves; truncated share strings; copy/paste character loss; unsupported future schema version (currently only testable synthetically, since only one schema version exists — named as a real limitation); resuming a saved draft where a previously-solved level was left mid-edit (solved flag must not survive stale); out-of-bounds occluder placement (needs an explicit legality/clipping decision); deleting occluders down to a mathematically unreachable target (must be caught by the save-time validator, not just "currently doesn't match"); NaN/Infinity values that survive JSON decode as well-typed garbage (caught by a numeric-sanity pass in the geometry validator, not just "did decode throw"); two occluders whose shadows overlap each other (must not double-count coverage); vault ID collision on import (always-new-entry, never silent overwrite); VoiceOver users needing full discrete equivalents for every gesture-driven action, not just labels.

**Data and privacy implications:** Fully local-only. The only data that ever leaves the device is the artifact string the user explicitly shares — pure puzzle geometry, no device or user identifiers. No accounts, no network calls, no analytics or tracking in this build. No permissions requested beyond the system Share Sheet. Clipboard is never read silently — only explicit user-chosen share/import actions touch it. The local save file gets swept into standard device/iCloud backup by default, which is fine and requires no special handling precisely because there's no PII in it — stated here as a deliberate, reviewed choice. Future pro-tier solve-history analytics stays local-only unless a later cloud-sync feature explicitly opts the user in, consistent with the standing cloud-ready seam.

**Risk register:** (1) Geometry/solve-check drift at the edges remains the top named risk since round one of this project — mitigation sharpened to two distinct, honestly-scoped checks instead of one overclaimed "bitwise" guarantee. (2) The version-mismatch import path can only be synthetically tested today since only one schema version exists — named as a real, unresolved test-coverage limitation, not solved. (3) The VoiceOver non-gesture control path is a locked requirement, but whether the same mutation functions can drive both gesture and discrete input without forking the data model is an architectural assumption that still needs confirmation in the design/architecture phase — flagged explicitly rather than assumed. (4) Live per-frame shadow recomputation performance is untested against occluder count or older supported devices — a laggy live shadow would undercut the exact tactile trust this app depends on. (5) Solved-status staleness — mitigated by never storing it as an independently driftable value. (6) Vault ID collision on import — mitigated by always-new-entry, never-overwrite.
