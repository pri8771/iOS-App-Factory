    *   `warmthRatioAtEntry: Double`
    *   `warmthStateAtEntry: String`

The repository wrapper [RelationshipRepository](file:///Users/pchordia/Documents/GitHub/nurture-ios/product_brief.md) will return immutable counterparts like `PersonDTO` and `NoteDTO` to ensure complete decoupling from SwiftData thread locks.

### 2. Navigation State & `@Observable` [AppRouter](file:///Users/pchordia/Documents/GitHub/nurture-ios/design_handoff.md)

@Observable @MainActor

*   **Add Person** runs in a `.sheet` driven by `presentAddPerson`.
*   **Paywall** runs in a `.sheet` driven by `presentPaywall`, receiving the context-specific `paywallSource` to display targeted upgrade copy.

### 3. Pure scoring via [WarmthCalculator](file:///Users/pchordia/Documents/GitHub/nurture-ios/app_features.md)
To prevent drift, the relationship scoring calculations must live inside a non-isolated `WarmthCalculator` struct using calendar-safe date arithmetic. This calculator takes a `PersonDTO`, `NoteDTO` list, and a target evaluation date, returning:
*   A decay ratio: `elapsedDays / expectedDays`.
*   A semantic state (`warm`, `cooling`, `cold`).
*   A localized, human-friendly explanation sentence (e.g., *"45 days since last check-in. You usually check in every 42 days."*).

By evaluating against an injectable reference date, the engine is fully decoupled from `Date()` and is testable under mock clocks.

### 4. App Group container URL Fallback
To ensure that future widgets or share extensions share access to the SwiftData store, the storage stack should look for a shared App Group path but fall back gracefully to the standard local container:

```swift
struct StorageDirectoryProvider {
    static func resolveDatabaseURL() -> URL {
        let appGroupId = "group.com.pchordia.nurture"
        if let containerURL = FileManager.default.containerURL(forSecurityApplicationGroupIdentifier: appGroupId) {
            return containerURL.appendingPathComponent("Nurture.store")
        }
        // Fallback for Simulator/No-provisioning environments
        let paths = FileManager.default.urls(for: .applicationSupportDirectory, in: .userDomainMask)
        return paths[0].appendingPathComponent("Nurture.store")
    }
}
```
This enables development to compile and run on simulator and personal developer teams without failing on provisioning mismatch.

### 5. Resilient StoreKit 2 & Offline Behavior
The custom Paywall must be resilient to network failures:
*   **entitlement checks**: Expose entitlements dynamically using `Transaction.currentEntitlements` mapped to an `EntitlementService` protocol.
*   **Product Loading Fallback**: If fetching product configurations from the App Store fails, we degrade gracefully by loading offline prices ($4.99/month, $39.99/year) as static fallbacks so the Paywall can still be dismissed and parsed.
*   **StoreKit testing**: Integrate a headless test suite using Apple's `StoreKitTest` and `SKTestSession` to verify purchase, expiration, and restoration loops.

### 6. Dependency Injection & Test Seams
Rather than using heavy DI frameworks, we will use a simple, lightweight environment pattern to inject services:

```swift
struct AppDependencies {
    let dateProvider: DateProvider
    let authService: LocalAuthenticationService
    let notificationService: NotificationService
    let entitlementService: EntitlementService
}
```

This allows us to substitute mocks for [DateProvider](file:///Users/pchordia/Documents/GitHub/nurture-ios/app_features.md) and [LocalAuthenticationService](file:///Users/pchordia/Documents/GitHub/nurture-ios/app_features.md) during unit testing, ensuring tests are independent of live device locks and actual time elapsed.
