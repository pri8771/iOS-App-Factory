## Final Output

**Lane-by-lane responsibilities**
- **data_domain** owns the `LongwaveSimulation` Swift package end to end: every domain type, the interference/throughput/economy math, `validate(_:)`, the closed-form `resolveOffline`, and the full deterministic test suite. Foundation-only, no SwiftUI/UIKit, no direct `Date()` calls. This is the frozen contract every other lane builds against — nobody else edits it, they request changes.
- **services_utilities** owns persistence (`SnapshotStoring`/`SnapshotStore`), commerce (`EntitlementProviding`/`StoreKitEntitlementService`/`ResolveRegionAccess`, the `.storekit` config, `PrivacyInfo.xcprivacy`), history retention, and the real-`FileManager`/real-StoreKit-config integration test suite.
- **primary_ui** owns the Xcode project itself (scaffolded once via file-system-synchronized groups so nobody hand-edits `project.pbxproj` after), `NetworkEngine`/`NetworkActioning`/`PresentedSurface`, app lifecycle wiring, the Canvas render/hit-testing layer, and all five screens. Becomes the standing integrator for every milestone merge into `main` starting the moment the project exists.
- **polish_resilience** owns VoiceOver custom actions, recovery-message copy, the Dynamic Type/non-color-signaling audit, final hardening (stress re-run, atomicity, performance), and the scripted first-session walkthrough that gates ship.

**Forbidden edit zones**
- `LongwaveSimulation/Sources/**` — data_domain only; other lanes consume, never patch.
- `Longwave.xcodeproj`, `Longwave/App/LongwaveApp.swift`, `Longwave/Engine/NetworkEngine.swift`, `Longwave/Engine/PresentedSurface.swift` — primary_ui only from the moment T-009 merges; anything else needs a named handoff request.
- `Longwave/Services/**` (persistence + commerce) — services_utilities only.
- Any polish_resilience fix that touches a primary_ui-owned view file goes in as a named patch the primary_ui owner reviews and merges — never a direct commit.

**Branching and merge strategy**
Lane branches feed a shared `longwave/integration` branch, not `main` directly. data_domain and services_utilities can both merge into `integration` independently the moment their own tests are green, since they share zero files. From T-009 onward, primary_ui is the standing integrator: it owns merging `integration` into `main`, milestone by milestone, gated on that milestone's acceptance criteria being green — never on "it compiles."

**Communication protocol**
Every task posts one sentence on completion stating what downstream lanes can now rely on, plus any fixture updates. Contract changes (a signature, a file another lane depends on) get proposed as a diff before editing — silent drift isn't allowed.

**Testing responsibilities**
data_domain proves simulation truth (continuity, offline math, invariants, first-session fixtures). services_utilities proves disk/entitlement truth (20-cycle stress, quarantine/rollback, four-state restore matrix). primary_ui proves contract/rendering truth (one source of derived state, `.engineNotReady`, idempotent milestone latching). polish_resilience proves the last-mile stuff that's easy to hand-wave (VoiceOver routing, Dynamic Type, color redundancy, performance, the walkthrough).
