This needs to be confirmed in code before submission, not assumed. If it isn't already set, it's a one-line fix; if the fix causes recognition to silently degrade on devices without on-device speech support, that degradation needs a user-facing explanation, not a silent failure.

Given that unresolved, I don't think the privacy label can be finalized with full confidence yet, but here's what I'd draft conditional on that being fixed: this app should be able to answer "Data Not Collected" for nearly every category, since there's no backend, no account system, no analytics SDK mentioned anywhere in the build transcript, and StoreKit purchase data is handled under Apple's own umbrella rather than the developer's.
